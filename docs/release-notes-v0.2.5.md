# Firestone2Green v0.2.5 更新日志

发布日期：2026-07-14

> 本版本将图形界面重构为 WPF + .NET 10，同时保持现有视觉风格、功能层级和核心修复能力不变，并集中修复布局对齐与运行日志持续撑高窗口的问题。

## WPF + .NET 10 重构

- 使用 WPF 和 .NET 10 重构图形界面，发布目标为 Windows x64 单 EXE；最终下载物包含一个轻量启动器和内嵌的 WPF 主程序。
- 保持暖米色、珊瑚色与深色日志区域组成的现有视觉风格，不增加多余强调色。
- 保持左侧功能优先级：
  1. **一键重启并授权**
  2. **持续化与打包**
  3. **精细控制**
- GitHub 与 QQ 群入口位于左侧，版本号 `v0.2.5` 位于右侧。
- 恢复程序启动时的环境、路径、权限和资源状态信息，方便用户自行判断当前状态。

## 布局与日志修复

- 继续适配 `560×420` 起的小窗口、常用桌面分辨率以及 100%～200% DPI 缩放。
- 统一卡片底部、按钮高度、行距与左右栏对齐关系。
- 固定“精细控制”的四个按钮位置，窗口增高时不再随剩余空间向下移动。
- 将“运行日志”恢复为固定视口的内部滚动窗口，日志追加时不再持续撑高整个软件。
- 日志支持内部上下滚动，并在追加新日志后自动跟随到最新一行。
- 小窗口会精简次要说明文字并保持底栏紧凑，主要操作、日志和版本信息仍可访问。

## 流程精简

- 移除“跳过存储隔离”和“监控秒数”界面选项及关联按钮。
- 普通一键流程不再隔离 Firestone 缓存，也不再等待 TCP 监控计时，减少不必要的等待和用户疑问。
- 脚本内部仍保留必要的高级兼容能力，不影响核心授权、头像、hosts、`AuthOnlyOnline`、持续修复和静默启动功能。

## 运行环境检测

- 最终下载物仍然只有一个 `Firestone2Green_v0.2.5.exe`，不会要求用户同时下载 DLL、脚本或源码。
- 为了保持 EXE 体积接近旧版，程序使用 .NET 10 Desktop Runtime 框架依赖模式。
- 轻量启动器会在进入 WPF 主界面前检测 Windows x64 的 .NET 10 Desktop Runtime。
- 检测不到时不会让用户面对系统原始报错，而是显示说明、**前往官方下载**按钮和**重新检测**按钮。
- 官方下载页面：`https://dotnet.microsoft.com/zh-cn/download/dotnet/10.0`；请选择 Windows x64 的 **Desktop Runtime**，不要下载 SDK 或 ASP.NET Core Runtime。

## 验证结果

- WPF Release 构建通过，目标框架为 `net10.0-windows`。
- 框架依赖单文件发布验证通过，最终 EXE 约 1.04 MiB。
- PowerShell 核心脚本语法检查通过。
- XAML/XML 结构检查与 `git diff --check` 通过。
- 日志压力测试确认大量日志只在日志视口内部滚动，不再使主窗口无限变长。

## 升级说明

已经安装旧版持续修复的用户，升级后建议执行一次：

1. 以管理员身份运行 Firestone2Green v0.2.5。
2. 点击 **移除持续修复**。
3. 再点击 **安装持续修复**。

这样可以确保计划任务、桌面快捷方式和内嵌脚本均更新为当前版本。

## 发布文件

- `Firestone2Green_v0.2.5.exe`：推荐普通用户下载。最终文件约 `1.04 MiB`，下载物只有一个 EXE；启动时会自动检测 .NET 10 Desktop Runtime。
- `Firestone2Green_v0.2.5_dist.zip`：包含单 EXE、使用教程、当前更新日志、中英文 README 与许可证，方便保存和转发完整说明。
- `Source code (zip)` / `Source code (tar.gz)`：GitHub 根据 `0.2.5` 标签自动生成的完整源码归档。

下载后右键以管理员身份运行。如果提示缺少 .NET 10 Desktop Runtime，点击 **前往官方下载**，安装 Windows x64 Desktop Runtime 后返回程序点击 **重新检测**。

## 注意事项

- Firestone2Green 完全免费，仅通过项目 GitHub 发布。
- 如果 hosts 被安全软件保护，请保持管理员运行，并在安全软件中允许 Firestone2Green 与 PowerShell 修改 hosts；程序会在日志中给出可执行的处理建议。
- 本版本不修改 `D:\Code\DESIGN-claude.md` 的内容。
## 发布后修复

- 启动器使用 Windows 管理员权限清单，打开编译后会请求 UAC 确认，确认后以管理员权限启动主程序。

- 修复启动器对 .NET 10 Desktop Runtime 的检测条件：改为检查 .NET Desktop Runtime 官方框架清单文件，避免运行库已安装但仍误报缺失。
- 已重新验证 Windows x64 .NET 10.0.5 与用户目录 .NET 10.0.9 两种安装位置。
