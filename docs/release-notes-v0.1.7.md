# Firestone2Green v0.1.7 更新日志

发布日期：2026-07-07

> 本版本继续修复持续修复安装失败问题，并针对 GitHub issue #2 优化高 DPI / 150% 缩放下的界面显示与日志体验。

## 重点更新

### 1. 修复高 DPI / 150% 缩放下界面显示不全

修复部分用户在 150% 缩放、小屏幕或高分辨率笔记本环境下出现的界面问题：

- 左下角按钮显示不完整。
- 持续修复区域被挤压。
- 运行日志区域被挤到只剩一行。
- 部分界面内容超出窗口后无法查看。

新版调整：

- 主界面改为可滚动布局，窗口空间不足时可以滚动查看完整内容。
- 降低最小窗口尺寸要求，兼容更多高 DPI 屏幕。
- 避免日志区域被布局压缩成一行。
- 保持默认窗口下的完整卡片布局。

### 2. 优化运行日志滚动体验

- 运行日志控件从普通 `TextBox` 调整为 `RichTextBox`。
- 支持鼠标滚轮查看历史日志。
- 隐藏右侧白色系统滚动条，保持黑色日志区域的整体观感。
- 保留日志自动追加、错误解释和授权成功检测。

### 3. 增强 Task Scheduler / 任务计划程序服务修复

0.1.6 已经会检测并尝试启动 `Schedule` 服务，但部分用户的系统中该服务可能被优化工具、精简系统或注册表策略禁用，普通 `Start-Service` 仍然无法拉起。

0.1.7 增强为：

- 输出 `Schedule` 服务当前状态和启动模式，方便排查。
- 先尝试 `Start-Service Schedule`。
- 再尝试：

```text
sc.exe config Schedule start= auto
```

- 如果仍失败，尝试写入注册表恢复启动类型：

```text
HKLM\SYSTEM\CurrentControlSet\Services\Schedule
Start = 2
```

- 再次尝试启动 `Schedule` 服务。
- 如果仍无法运行，会明确提示用户重启电脑后再安装持续修复。

### 4. GUI 错误解释优化

- 修复部分 Task Scheduler 错误行被截断后，GUI 没有识别到具体原因，只显示泛化“退出码 1”的问题。
- 现在只要日志中出现 `Task Scheduler / 任务计划程序`、`任务计划程序服务未运行` 等关键词，都会显示对应中文解释。

## 修复的问题

- 修复 150% 缩放下界面显示不完整的问题。
- 修复日志区域在小窗口中被压缩到只剩一行的问题。
- 修复日志不能用鼠标滚轮查看历史输出的问题。
- 修复日志右侧白色滚动条破坏视觉风格的问题。
- 修复部分系统中 Task Scheduler 服务被禁用后，持续修复仍安装失败且提示不清楚的问题。
- 修复 Task Scheduler 相关错误被误解释成普通“退出码 1”的问题。

## 用户处理建议

如果用户遇到持续修复安装失败，并看到 Task Scheduler / 任务计划程序相关提示：

1. 下载并运行新版 `Firestone2Green.exe`。
2. 右键选择 **以管理员身份运行**。
3. 点击 **安装持续修复**。
4. 如果日志提示已经尝试恢复 `Schedule` 服务但仍失败，请先重启电脑。
5. 重启后再次管理员运行 Firestone2Green，并点击 **安装持续修复**。

如果重启后仍失败：

1. 按 `Win + R`。
2. 输入 `services.msc`。
3. 找到 **Task Scheduler / 任务计划程序**。
4. 将启动类型设置为 **自动**。
5. 点击 **启动**。
6. 回到 Firestone2Green 再次安装持续修复。

如果服务列表中找不到 Task Scheduler，说明系统可能被精简或组件损坏，需要先恢复 Windows 任务计划程序组件，否则无法使用持续修复和静默启动任务。

## 升级说明

如果之前安装过 **持续修复**，升级后建议重新安装一次：

1. 管理员运行新版 `Firestone2Green.exe`。
2. 点击 **移除持续修复**。
3. 点击 **安装持续修复**。
