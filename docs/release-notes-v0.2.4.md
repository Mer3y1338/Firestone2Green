# Firestone2Green v0.2.4 更新日志

发布日期：2026-07-11

> 本版本集中修复 Windows 高 DPI、窄窗口和小屏工作区下的界面挤压、裁切与底栏过厚问题，不改变授权、hosts、头像、网络和持续修复的核心逻辑。

## 响应式界面重构

- 界面会根据窗口逻辑尺寸和当前 DPI，在普通、紧凑、高缩放窄布局之间自动切换。
- 最小逻辑尺寸调整为 `560×420`，启动时会把窗口尺寸限制在当前显示器工作区内，避免窗口超出屏幕。
- 窄窗口会隐藏次要 Hero 内容和长提示，保留主要操作、状态和必要说明，减少拥挤。
- 主内容只进行纵向滚动，底栏固定在窗口底部；缩小界面时底栏高度更紧凑，不再出现过粗黑色区域。
- 左侧模块优先级和顺序固定为：
  1. **一键重启并授权**
  2. **持续化与打包**
  3. **精细控制**
- GitHub 按钮移动到底栏左侧，版本号 `v0.2.4` 固定在最右侧，使信息层级更清晰。
- 界面继续采用一体化卡片结构，不恢复额外标签页，也不增加新的强调色。

## 高 DPI 裁切修复

- 使用窗口实际 DPI 重新计算布局、间距、圆角、按钮文字留白和数字步进控件尺寸。
- 收到 `WM_DPICHANGED` 后重新约束窗口并应用响应式布局，改善窗口跨不同 DPI 显示器后的显示完整性。
- 修复 125%～200% 缩放下固定行高和嵌套表格未正确缩放的问题。
- 修复高缩放窄布局中“打开目录”“管理员重启”按钮底部轻微裁切的问题。
- 修复高 DPI 下“运行日志”标题行可能溢出父容器的问题。
- 自定义按钮、状态胶囊、数字步进器和卡片绘制均按 DPI 缩放，避免文字、阴影和边框被截断。

## 验证结果

- 已完成 100% 实际窗口几何测试，覆盖：
  - `1180×900`
  - `880×640`
  - `640×500`（顶部与滚动到底部）
  - `588×445`（顶部与滚动到底部）
  - `560×420`（顶部与滚动到底部）
- 已通过 125%、150%、175%、200% DPI 布局模拟审计。
- 审计确认关键控件无内部裁切、无横向滚动，底栏固定可见，左侧优先级与 GitHub / 版本号顺序正确。
- 以上结果不等同于在所有真实多显示器硬件组合上逐一测试。

## 保持不变

- v0.2.3 的 `F2G_ERROR`、`F2G_ERROR_TYPE`、`F2G_ERROR_LINE`、`F2G_ERROR_COMMAND` 原始异常定位保持不变。
- hosts 缺失、空白或只有 `hosts.txt` 时的自动恢复，以及动态路径、只读解除、临时 ACL 接管和恢复、备份、写后校验与失败回滚保持不变。
- 一键授权、头像修复、`AuthOnlyOnline` 网络恢复、路径搜索、持续修复、静默快捷方式和 GitHub 更新检查保持不变。
- `D:\Code\DESIGN-claude.md` 的内容和视觉规范未修改。

## 升级说明

本版本只修改 GUI 和版本信息，内置核心 PowerShell 功能逻辑没有新增行为。已经安装持续修复的用户可以直接升级；如果桌面快捷方式或计划任务本身异常，再执行一次：

1. 以管理员身份运行 `Firestone2Green_v0.2.4.exe`。
2. 点击 **移除持续修复**。
3. 再点击 **安装持续修复**。
