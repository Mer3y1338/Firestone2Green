# 使用教程：Firestone2Green

## 1. 首次使用

如果你只收到一个 `Firestone2Green_v???.exe`，可以直接使用。脚本和头像已经内置在 EXE 中，首次运行会自动释放到：

```text
%LOCALAPPDATA%\Firestone2Green\scripts\Firestone2Green.ps1
```

操作步骤：

1. 右键 `Firestone2Green_v???.exe`。
2. 选择 **以管理员身份运行**。
3. 先看 **路径设置**，如果没有自动识别 Overwolf 路径，点击 **自动搜索**；仍找不到时点击 **选择路径**，手动选择 Overwolf 根目录，也就是打开后能直接看到 `OverwolfLauncher.exe` 或 `Overwolf.exe` 的那一层目录。如果只安装了 Overwolf（狼头）但还没安装 Firestone，请先在 Overwolf 中安装并正常打开一次 Firestone。
4. 点击 **一键重启并授权**。
5. 等到日志最后出现：
   - `授权状态：hasPremium=True`
   - `shouldDisplayAds=False`
   - `头像已注入窗口`
   - `AuthOnlyOnline 已启用`
   - `退出码 0`

### 窄窗口和高 DPI 使用说明

- 左侧功能按 **一键重启并授权 → 持续化与打包 → 精细控制** 排列，优先保证推荐操作和持续修复可见。
- 在较窄窗口、较小屏幕或 125%～200% 缩放下，程序会缩短说明文字、隐藏次要 Hero 文案并压缩底栏高度；按钮、输入框、日志标题和版本信息仍保持完整。
- 主内容过长时只允许纵向滚动，底栏会固定在窗口底部；底栏左侧是 GitHub，版本号固定在最右侧。
- 运行日志使用固定视口和内部滚动条；日志持续追加时不会再撑高整个窗口，并会自动跟随到最新一行。
- 程序会把初始尺寸和最小尺寸限制在当前显示器工作区内，跨不同 DPI 显示器后会重新应用布局。
- v0.2.8 保持当前 WPF 界面不变，并沿用已完成的 100% 实际窗口几何测试与 125%～200% DPI 布局模拟审计；未宣称在所有真实多显示器硬件组合上逐一测试。

## 3. 持久化

Firestone 的授权状态是运行时状态，电脑重启、Firestone 完全退出或 Overwolf 更新后可能会恢复默认。Firestone2Green 的持久化方式是“后台自动补授权”：

1. 管理员运行 `Firestone2Green_v???.exe`。
2. 点击 **安装持续修复**。
3. 程序会安装两个计划任务并启动后台监听，但不会主动启动或重启 Firestone：
   - `Firestone2Green`：登录后常驻监听 Firestone 手动启动事件；检测到 `-launchapp` 后只探测当前有效 Automation 端口，成功时补授权，失败时只维持网络规则，不会反复重启 Firestone。
   - `Firestone2Green Launch`：按需静默启动 Firestone 并补授权。
4. 桌面会出现 **Firestone2Green 启动 Firestone** 快捷方式。
5. 需要立刻授权时点击 **一键重启并授权**，以后用这个快捷方式启动 Firestone，不会弹 PowerShell 窗口，也不会弹完成提示。

如果仍然使用原始 Firestone 图标启动，事件监听任务会在 `-launchapp` 启动事件出现时触发；若没有有效 Automation 接口，只会维持网络规则并等待下一次启动，不会结束或反复重启 Firestone。为了稳定自动授权，仍然推荐固定使用新快捷方式。

> 新版会先尝试 v0.2.7 已验证的 `-launchapp ... --automation 18765 --enable-automation` 兼容启动；只有默认端口被占用或兼容启动未通过 `pingServer` 验证时，才会自动尝试 `18766`～`18770`。尚未启动 Firestone 时，备用流程会先启动 runtime、紧接着发送一次标准启动命令，再验证端口；兼容启动请求已经发出时不会再次启动 Firestone。程序不会结束端口占用进程，也不会反复重启 Firestone。升级后管理员运行新版并点击一次 **一键重启并授权**，程序会安全刷新已安装的旧 Firestone2Green 监听器。

## 4. Firestone 更新后怎么办

1. 重新右键管理员运行 `Firestone2Green_v???.exe`。
2. 点击 **一键重启并授权**。
3. 若只需要补头像/补授权，点击 **刷新授权+头像**。

## 5. 头像说明

头像已经内置进 PowerShell 脚本和 EXE。修复逻辑只允许命中左下角登录/账户区域，不会替换酒馆战旗英雄头像、环境套牌图标、套牌图标或卡牌图片。

## 6. 常见问题

### 提示不是管理员

关闭工具，右键 `Firestone2Green_v???.exe`，选择“以管理员身份运行”。

### 日志显示退出码 1

从 v0.2.3 开始，查看退出码上方的 `F2G_ERROR`、`F2G_ERROR_TYPE`、`F2G_ERROR_LINE` 和 `F2G_ERROR_COMMAND`，它们分别表示真正的异常、异常类型、原始代码行和命令。需要完整信息时点击 **打开报告**，查看时间最新的 `FirestoneOfflineReport_*.json`。

### hosts 不存在、显示 hosts.txt 或提示被保护

无需手工寻找或下载 hosts 文件。新版会自动：

1. 从 Windows 注册表读取实际 hosts 目录，并兼容 32 位进程的系统目录重定向。
2. 没有无扩展名 `hosts` 时自动创建；`hosts` 不存在、为零字节或只有空白内容时，先写入 Windows 默认 HOSTS 模板，再加入 Firestone2Green 阻断段。
3. 如果只有 `hosts.txt`，复制其内容为 `hosts` 并保留原文件；如果 `hosts.txt` 也是空的，目标 `hosts` 会恢复默认模板。已有非空内容不会被覆盖。
4. 自动解除只读，短时占用重试 3 次；必要时临时修复文件 ACL，完成后恢复原 ACL/所有者。
5. 修改前备份到 `%LOCALAPPDATA%\Firestone2Green\hosts-backups`，只保留最近 10 份；写入后重新校验，失败时尽量回滚。

v0.2.8 起，如果日志仍出现 `HOSTS_PROTECTION_ACTIVE`、`HOSTS_FILE_BUSY`、`HOSTS_CREATE_FAILED` 或 `HOSTS_WRITE_VERIFY_FAILED`，无需关闭火绒等安全软件：程序会回滚 hosts，并自动切换到 Windows 防火墙精确域名阻断。支持动态 FQDN 规则；旧系统会自动改用解析后的 IP 规则。备用规则创建成功后，后续启动会直接复用，不再反复写 hosts。只有 hosts 与防火墙都被禁止时才会显示 `UnprotectedRuntime`，此时本地启动和授权仍继续，但日志会明确说明精确端点阻断未生效。不要自行修改 ACL，也不要购买或下载所谓 hosts 修复文件。

### 点击“刷新授权+头像”失败

说明 Firestone 当前不是通过 automation 端口启动的。点击 **一键重启并授权**。

如果报告显示 `launchResult=LaunchOnlyDegraded`，表示 Firestone 启动请求已经发送，但 v0.2.7 兼容路径和 `18765`～`18770` 的有限候选端口都没有通过标准 `pingServer` 验证。程序不会把这种情况误报为退出码 `1`：网络会保持 `AuthOnlyOnline`，本次只跳过运行时补授权。程序不会结束 `NetHost.exe`、System 或任何未知端口占用进程，也不需要用户手工选择端口。管理员运行新版并再点一次 **一键重启并授权** 即可重试。

### 数据/套牌功能无法使用

点击 **恢复全功能网络**，再点击 **验证状态**。报告中应显示 `NetworkMode=AuthOnlyOnline` 且 `NetworkPolicyOK=True`。hosts 可写时 `ExactDomainMethod=Hosts`；hosts 被保护时会显示 `DynamicFqdnFirewall` 或 `ResolvedIpFirewall`，此时出现精确域名备用防火墙规则属于正常现象。

### 头像没有变化

保持 Firestone 主窗口打开，再点一次 **刷新授权+头像**。报告中应出现 `头像已注入窗口` 或 `currentTargetMatches=1`，且 `leakedAvatarUrlMatches=0`。
### 套牌信息时有时无

新版启动流程会在 Firestone 启动前先切到 `AuthOnlyOnline`，避免套牌/环境数据在启动断网窗口里初始化失败。若仍然遇到，点击 **一键重启并授权** 或通过桌面 **Firestone2Green 启动 Firestone** 快捷方式启动。
