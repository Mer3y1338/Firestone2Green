# 使用教程：Firestone2Green

## 1. 首次使用

如果你只收到一个 `Firestone2Green.exe`，可以直接使用。脚本和头像已经内置在 EXE 中，首次运行会自动释放到：

```text
%LOCALAPPDATA%\Firestone2Green\scripts\Firestone2Green.ps1
```

操作步骤：

1. 右键 `Firestone2Green.exe`。
2. 选择 **以管理员身份运行**。
3. 先看 **路径设置**，如果没有自动识别 Overwolf 路径，点击 **自动搜索**；仍找不到时点击 **选择路径**，手动选择 Overwolf 根目录，也就是打开后能直接看到 `OverwolfLauncher.exe` 或 `Overwolf.exe` 的那一层目录。
4. 点击 **一键重启并授权**。
5. 等到日志最后出现：
   - `授权状态：hasPremium=True`
   - `shouldDisplayAds=False`
   - `头像已注入窗口`
   - `AuthOnlyOnline 已启用`
   - `退出码 0`

## 2. 按钮说明

| 按钮 | 用途 |

| 更新检查 | 程序启动后自动连接 GitHub Releases 对比版本；无更新会在右上角显示“已是最新”，网络失败会弹窗提示。 |
| 错误解释 | 运行日志出现常见错误时，会自动追加几句原因和处理建议。 |
| 选择路径 | 手动选择 Overwolf 根目录；正确目录必须直接包含 `OverwolfLauncher.exe` 或 `Overwolf.exe`。如果选到子目录或上级目录，会弹窗提示正确位置。 |
| 一键重启并授权 | 推荐按钮。先重启 Firestone，完成授权、头像替换，然后恢复全功能网络。 |
| 刷新授权+头像 | Firestone 已经打开且 automation 端口可用时使用，不重启。 |
| 恢复全功能网络 | 移除程序级防火墙阻断，只保留必要的上报端点阻断。 |
| 验证状态 | 生成状态报告，查看完整性、网络模式、hosts 和进程连接。 |
| 安装持续修复 | 安装手动启动事件监听任务和桌面快捷方式；不会主动启动 Firestone，以后只有检测到 Firestone 被手动启动时才自动补授权。 |
| 移除持续修复 | 删除上面的计划任务。 |
| 打开报告 | 打开 JSON 报告目录。 |
| 打开目录 | 打开工具目录。 |

## 3. 持久化

Firestone 的授权状态是运行时状态，电脑重启、Firestone 完全退出或 Overwolf 更新后可能会恢复默认。Firestone2Green 的持久化方式是“后台自动补授权”：

1. 管理员运行 `Firestone2Green.exe`。
2. 点击 **安装持续修复**。
3. 程序会安装两个计划任务并启动后台监听，但不会主动启动或重启 Firestone：
   - `Firestone2Green`：登录后常驻监听 Firestone 手动启动事件；只有检测到 `-launchapp` 启动 Firestone 且未开启 automation 时，才会立即静默重启并补授权。
   - `Firestone2Green Launch`：按需静默启动 Firestone 并补授权。
4. 桌面会出现 **Firestone2Green 启动 Firestone** 快捷方式。
5. 需要立刻授权时点击 **一键重启并授权**，以后用这个快捷方式启动 Firestone，不会弹 PowerShell 窗口，也不会弹完成提示。

如果仍然使用原始 Firestone 图标启动，事件监听任务会在 `-launchapp` 启动事件出现时触发；若 automation 端口不可用，会静默重启并补授权。它不会在你未启动 Firestone 时主动拉起。为了体验最快，仍然推荐固定使用新快捷方式。

## 4. Firestone 更新后怎么办

1. 重新右键管理员运行 `Firestone2Green.exe`。
2. 点击 **一键重启并授权**。
3. 若只需要补头像/补授权，点击 **刷新授权+头像**。

## 5. 头像说明

头像已经内置进 PowerShell 脚本和 EXE。修复逻辑只允许命中左下角登录/账户区域，不会替换酒馆战旗英雄头像、环境套牌图标、套牌图标或卡牌图片。

## 6. 常见问题

### 提示不是管理员

关闭工具，右键 `Firestone2Green.exe`，选择“以管理员身份运行”。

### 点击“刷新授权+头像”失败

说明 Firestone 当前不是通过 automation 端口启动的。点击 **一键重启并授权**。

### 数据/套牌功能无法使用

点击 **恢复全功能网络**，再点击 **验证状态**。报告中应显示 `NetworkMode=AuthOnlyOnline`，防火墙规则数应为 `0`。

### 头像没有变化

保持 Firestone 主窗口打开，再点一次 **刷新授权+头像**。报告中应出现 `头像已注入窗口` 或 `currentTargetMatches=1`，且 `leakedAvatarUrlMatches=0`。
### 套牌信息时有时无

新版启动流程会在 Firestone 启动前先切到 `AuthOnlyOnline`，避免套牌/环境数据在启动断网窗口里初始化失败。若仍然遇到，点击 **一键重启并授权** 或通过桌面 **Firestone2Green 启动 Firestone** 快捷方式启动。
