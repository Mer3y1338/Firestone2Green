# Firestone2Green v0.2.2 更新日志

发布日期：2026-07-10

> 本版本重点重构界面适配与 hosts 自动修复流程，减少用户在小窗口、高 DPI、特殊 hosts 路径、只读或受保护 hosts 环境下的手动处理。

## 重点更新

### 1. 一体化界面与多分辨率适配

- 移除 Windows 原生标题栏，改为与画布同色的一体化窗口控制区，不再显示额外的软件标题标签。
- 保留窗口拖动、四边与四角缩放、双击最大化、最小化、最大化/还原和关闭操作。
- 左侧功能按使用优先级固定为：**一键重启并授权 → 持续化与打包 → 精细控制**。
- 窄窗口自动切换为单列并启用整体滚动，避免按钮、说明、日志和底栏被裁切。
- 小窗口底栏删减非必要说明并降低高度，GitHub 按钮移到左侧，版本号固定在最右侧。
- 修复高 DPI、窗口缩放及双列/单列切换时的显示不完整问题。

### 2. 视觉样式统一

- 按 `DESIGN-claude.md` 收敛颜色体系，减少无意义的状态色、渐变、阴影和装饰色。
- 珊瑚色仅用于主操作、按下状态和已选择复选框；普通按钮与状态信息统一使用中性色。
- 普通按钮统一为纯色背景、细边框和紧凑圆角，提高界面整体性。
- 重构“跳过缓存隔离”控件，修复部分缩放倍率下显示异常的问题。

### 3. hosts 路径与缺失文件自动处理

- 不再写死 `C:\Windows\System32\drivers\etc\hosts`，优先读取 Windows `Tcpip\Parameters\DataBasePath`。
- 兼容 32 位进程访问 64 位系统目录时的 `Sysnative` 重定向。
- hosts 目录或无扩展名 `hosts` 文件不存在时自动创建。
- 只有 `hosts.txt` 时自动复制其内容创建 `hosts`，并保留原文件；不会误处理 `hosts.ics`。

### 4. hosts 保护与文件占用恢复

- 写入前自动解除只读属性，并对短时间文件占用进行重试。
- 常规管理员权限仍被阻止时，临时接管 hosts，并只向当前管理员 SID 授予必要写权限。
- 操作完成后精确恢复原 ACL 和所有者，不会永久向 `Everyone` 或 `Users` 开放权限。
- 修改前自动备份到 `%LOCALAPPDATA%\Firestone2Green\hosts-backups`，最多保留最近 10 份。
- 保留用户原有 hosts 条目和非 ASCII 注释字节，只维护 Firestone2Green 自己的标记段。
- 写入后重新读取校验，失败时尝试回滚，降低安全软件或 hosts 保护导致的损坏风险。

### 5. 错误说明与自助恢复

新增并优化以下错误的中文自助说明：

- `HOSTS_PROTECTION_ACTIVE`
- `HOSTS_FILE_BUSY`
- `HOSTS_CREATE_FAILED`
- `HOSTS_WRITE_VERIFY_FAILED`
- hosts 访问被拒绝
- Overwolf 启动器或安装根目录未找到
- PowerShell 执行策略或安全软件拦截

程序会尽量给出可能原因和下一步操作，减少用户需要手动询问路径或 hosts 权限问题的情况。

## 升级说明

如果以前安装过 **持续修复**，升级到 v0.2.2 后请执行一次：

1. 以管理员身份运行 `Firestone2Green_v0.2.2.exe`。
2. 点击 **移除持续修复**。
3. 再点击 **安装持续修复**。

这样计划任务、桌面快捷方式和后台脚本才会更新为 v0.2.2 的 hosts 修复逻辑。
