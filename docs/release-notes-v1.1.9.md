# Firestone2Green v0.1.8 更新日志

发布日期：2026-07-07

> 本版本继续跟进 GitHub issue #2 的高 DPI 界面反馈，并规范发布产物命名。之后生成的 EXE / ZIP 文件名会自带版本号，方便用户区分版本。

## 重点更新

### 1. 发布文件名增加版本号

从本版本开始，构建输出文件名会带版本号：

```text
Firestone2Green_v1.1.9.exe
Firestone2Green_v1.1.9_dist.zip
```

这样用户下载、转发或排查问题时，可以直接从文件名确认版本，避免不同版本混用。

### 2. 修复 150% 缩放下顶部大标题被裁切

根据 GitHub issue #2 的后续反馈，150% 缩放下按钮和日志已经正常，但顶部 `Firestone2Green` 大标题会被下方副标题裁掉一部分。

本版本调整：

- 加高顶部 Hero 区域。
- 加高大标题行和副标题行。
- 禁止大标题被省略裁切。
- 确保 150% 缩放下顶部标题完整显示。

### 3. 修复隐藏日志滚动条后无法滚轮滚动

此前为了美观隐藏了日志右侧白色滚动条，但部分环境下会导致鼠标滚轮无法滚动历史日志。

本版本调整：

- 保持日志右侧滚动条隐藏。
- 增加自定义鼠标滚轮滚动逻辑。
- 继续支持查看历史日志。
- 日志追加时仍自动滚到底部。

### 4. 拖动窗口性能优化

针对用户反馈的“拖动边框有点卡”，本版本做了低风险优化：

- 拖动窗口边框时临时暂停主内容区连续重绘。
- 拖动过程中跳过高频布局计算。
- 松开边框后一次性恢复布局并刷新。
- 降低高 DPI 下卡片阴影、按钮和日志控件重复绘制开销。

## 修复的问题

- 修复顶部 `Firestone2Green` 大标题在 150% 缩放下显示半截的问题。
- 修复隐藏日志滚动条后，鼠标滚轮无法查看历史日志的问题。
- 优化拖动窗口边框时的卡顿问题。
- 修复发布文件名不带版本号，导致用户难以区分版本的问题。

## 升级说明

如果之前安装过 **持续修复**，升级后建议重新安装一次：

1. 管理员运行新版 `Firestone2Green_v1.1.9.exe`。
2. 点击 **移除持续修复**。
3. 点击 **安装持续修复**。

```text
Firestone2Green_v1.1.9.exe
```
